# Продольно-поперечная: текущий статус

Дата фиксации: 2026-05-06

## Статус

Режим `Продольно-поперечная` (`longitudinal`) считаем исправленным и закрытым по текущему дефекту.

## Что считаем правильным поведением

- Режим строит регулярную ортогональную сетку фрагментов внутри зоны.
- Контур зоны клипует сетку, но не должен создавать локальную "вторую" сетку на сложных бортах.
- При одинаковых входных параметрах результат детерминирован.
- Для режима не допускаются случайные смещения разрезов (`jitter`) и скрытая вариативность.

## Что зафиксировано как достигнутый результат

- Проблема визуально странного поведения правого края в `Продольно-поперечной` закрыта рабочей правкой.
- Режим больше не находится в активной доработке.
- Повторно менять алгоритм режима без нового отдельного дефекта не нужно.

## Правило дальнейшей работы

- Если появляется новый дефект по `Продольно-поперечной`, он заводится и разбирается как новый кейс.
- До появления нового отдельного дефекта режим считаем стабильным.
