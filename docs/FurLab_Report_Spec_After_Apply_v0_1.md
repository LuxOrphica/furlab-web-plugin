# FurLab — Спецификация отчёта после Apply (v0.1)

## 1. Назначение
После подтверждения выкладки (кнопка `Применить`) оператор получает отчёт по результату раскладки для проверки, печати и экспорта.

## 2. Точка входа
- Кнопка: `Отчёты`
- Доступность: только если есть применённая выкладка (`applied=true`) и хотя бы один итоговый фрагмент.

## 3. Состав окна отчёта
- Заголовок: `Спецификация (Модель <id>)`
- Вкладки по деталям: `Деталь 1`, `Деталь 2`, ...
- Фильтр детали/зоны (справа сверху)
- Левая часть: таблица фрагментов выбранной детали/зоны
- Правая часть: мини-схема детали с нумерацией фрагментов
- Верхние действия: `Экспорт`, `Печать`, `Закрыть`

## 4. Таблица (обязательные колонки)
- `Рис.` — миниатюра формы фрагмента
- `Направление ворса`
- `Фрагмент` (ID в формате `detail-zone-index`)
- `Меховой материал`
- `Кол-во, шт`
- `Площадь, мм²`
- `Инвентарный номер` (inventoryTag) — только для инвентарных режимов

## 5. Данные отчёта
Источник: только применённый результат текущей выкладки.

Для каждой записи:
- `fragmentId`
- `detailId`
- `zoneId`
- `materialName`
- `inventoryTag` (если режим инвентарный)
- `napDirectionDeg`
- `areaMm2`
- `count`
- `fragmentContour` (для миниатюры/схемы)

Сводно:
- общее число фрагментов
- суммарная площадь по детали/зоне
- покрытие по детали/зоне

## 6. Правила
- До `Применить` отчёт недоступен.
- Изменения ручной раскладки после Apply требуют повторного Apply для обновления отчёта.
- Отчёт строится по итоговым фрагментам результата, а не по исходным полным контурам кусков.

## 7. Экспорт и печать
- Экспорт: `CSV` и `PDF` (минимум CSV обязателен в v0.1).
- Печать: текущая деталь или все детали.

## 8. Acceptance criteria
1. Кнопка `Отчёты` появляется/активна только при `applied=true`.
2. Для выбранной детали отображаются корректные строки по фрагментам.
3. Схема справа соответствует строкам таблицы (ID/нумерация совпадают).
4. `Экспорт CSV` содержит те же поля, что таблица.
5. Переход по вкладкам деталей не сбрасывает фильтр зоны без явного действия пользователя.

## 9. Версионирование
- Текущая версия: `v0.1`
- Все изменения формата отчёта фиксируются в changelog документа.
