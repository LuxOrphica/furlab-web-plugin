# FurLab — Геометрическое ядро

Документация математических алгоритмов, реализованных в системе автоматического раскроя меха FurLab.

---

## Архитектура

```
Входные данные
  контуры деталей (CLO 3D) · зоны · материал · ворс
          │
          ▼
  Фрагментация зоны          ← server.js + src/modes/
  продольная / диагональная / ёлочка / радиальная
          │
          ▼
  Подготовка деталей         ← piece_working_area.js
  упрощение (RDP)            ← inventory_solver_candidates.js
  эрозия на шов              ← piece_working_area.js
          │
          ▼
  Солвер укладки             ← cover_grid_solver.js
  растеризация · битсеты     ← solver_primitives.js
  подбор позиций             ← inventory_direct_solver.js
          │
          ▼
  Булева геометрия           ← polygon_ops.js
  union · intersect · diff · area
          │
          ▼
  Оценка и диагностика       ← inventory_solver_nap.js
  отклонение ворса · метрики ← inventory_solver_diagnostics.js
```

Внешняя зависимость для булевых операций — библиотека
[polygon-clipping](https://github.com/mfogel/polygon-clipping)
(алгоритм Martínez–Rueda–Feito, 2009).
Для операций offset/inset используется
[Clipper](http://www.angusj.com/delphi/clipper.php) (Angus Johnson).

## 1. Булева геометрия (`polygon_ops.js`)

### Представление данных

Полигон хранится в формате **GeoJSON MultiPolygon**:

```
MultiPolygon = Polygon[]
Polygon      = Ring[]        // кольцо[0] — внешний контур, кольцо[i>0] — дырки
Ring         = [x, y][]      // замкнутое: последняя точка == первая
```

Входные контуры — массив `{x, y}` объектов — конвертируются через `pointsToMultiPolygon`.

### Площадь: формула Гаусса (Shoelace)

Для кольца из n вершин:

```
S = ½ · |Σᵢ (xᵢ · yᵢ₊₁ − xᵢ₊₁ · yᵢ)|,   индексы по модулю n
```

Для полигона с дырками:
```
S(poly) = S(outer_ring) − Σ S(hole_i)
```

Для MultiPolygon: `S = Σ S(polyᵢ)`.

**Реализация:** `ringAreaAbs()`, `multiPolygonArea()`.

### Булевы операции

| Функция | Операция | Свойства |
|---|---|---|
| `unionMulti(A, B)` | A ∪ B | S(A∪B) = S(A) + S(B) − S(A∩B) |
| `intersectMulti(A, B)` | A ∩ B | S(A∩B) ≤ min(S(A), S(B)) |
| `diffMulti(A, B)` | A \ B | S(A\B) = S(A) − S(A∩B) |

**Ключевое свойство (сохранение площади):**
```
S(A) = S(A ∩ B) + S(A \ B)
```
Проверено тестами для произвольных перекрывающихся квадратов.

### Численная стабильность

Координаты округляются до 6 знаков после запятой перед передачей в `polygon-clipping`.
При сбое булевого ядра на патологическом контуре (`safeOp`) операция возвращает
исходный операнд вместо исключения.

## 2. Эрозия и расширение контура (`piece_working_area.js`)

### Постановка задачи

При ручном подборе детали кроя накладывается на фрагмент зоны.
Деталь должна перекрывать фрагмент своим **ядром** — контуром,
уменьшенным на величину припуска на шов `reserveMm`.
Ядро = деталь без шовной зоны.

### Математика: Минковская эрозия

Для полигона P и круга B(r):

```
Erosion(P, r) = P ⊖ B(r)
```

Геометрически — сдвиг каждого ребра внутрь на r, пересоединение углов
по типу `JoinType.jtRound` (ClipperLib).

**Реализация:** `insetPathBest(path, reserveMm, scale)` —
вызывает `ClipperOffset.Execute(out, -reserve * scale)`.

### Масштабирование

ClipperLib работает с целыми числами.
Все координаты умножаются на `scale = 1000` (т.е. 1 мм → 1000 единиц Clipper),
что даёт субмиллиметровую точность.

### Бинарный поиск максимального допустимого отступа

Если запрошенный `reserveMm` схлопывает геометрию (деталь слишком мала),
выполняется поиск максимально возможного отступа:

```
lo = 0,  hi = reserveMm
for 14 iterations:
    mid = (lo + hi) / 2
    if inset(mid) is valid:  lo = mid, best = mid
    else:                    hi = mid
```

14 итераций дают точность `reserveMm / 2¹⁴ ≈ 0.006%`.

**Статусы:** `ok` · `reserve_clamped` · `reserve_too_large` · `invalid_input` · `clean_failed`.

### Расширение контура (outset)

`outsetPath(points, reserveMm)` — симметричная операция: `Execute(out, +reserve * scale)`.
Используется при визуализации шовных зон.

**Свойство:** `S(outset(inset(P, r), r)) ≤ S(P)` — скруглённые входящие углы
не восстанавливаются при обратном расширении.

## 3. Упрощение контура: алгоритм Рамера–Дугласа–Пекера (`inventory_solver_candidates.js`)

### Зачем

Контуры деталей из CLO 3D содержат сотни точек.
Солвер перебирает тысячи позиций — упрощение до ~90 точек
ускоряет геометрические проверки в 5–10 раз.

### Алгоритм (Ramer–Douglas–Peucker, 1972)

Вход: полилиния `[p₀, p₁, …, pₙ]`, порог `ε`.

```
RDP(input, ε):
  keep[0] = keep[n] = true
  stack = [(0, n)]
  while stack not empty:
    (s, e) = stack.pop()
    найти i* = argmax_{s<i<e} dist(pᵢ, segment(pₛ, pₑ))
    если dist(pᵢ*, pₛpₑ) > ε:
      keep[i*] = true
      stack.push((s, i*), (i*, e))
  return [pᵢ : keep[i]]
```

**Расстояние от точки до отрезка:**

```
d(P, AB) = |AP × AB| / |AB|

где × — скалярное значение псевдо-кросс-произведения в 2D:
AP × AB = (P−A) × (B−A) = (Px−Ax)(By−Ay) − (Py−Ay)(Bx−Ax)
```

В реализации используется проекция с зажимом `t = clamp((AP·AB)/|AB|², 0, 1)`.

### Гарантированное свойство

```
∀ p ∈ input : dist(p, simplified) ≤ ε
```

### Адаптивный ε

Начальный `ε = max(0.05, diag × 0.0015)` где `diag` — диагональ bounding box.
При превышении лимита точек `ε` умножается на 1.6 до 10 раз.
Финальный fallback — равномерная прореживание через `step`.

## 4. Угловое отклонение ворса (`inventory_solver_nap.js`)

### Постановка

Мех имеет направление ворса. Деталь кроя можно уложить
на фрагмент только если отклонение ворса детали от целевого
направления зоны не превышает порога `napToleranceDeg`.

### Нормализация угла

```
wrapSignedDeg(v):
  x = v mod 360
  if x > 180:  x -= 360
  if x ≤ -180: x += 360
  return x          // результат ∈ (-180, 180]
```

### Отклонение ворса

```
computeNapDeviation(targetDeg, rotatedDeg, allowFlip):
  d₀   = wrapSignedDeg(rotatedDeg − targetDeg)
  if not allowFlip: return d₀
  d₁₈₀ = wrapSignedDeg((rotatedDeg + 180) − targetDeg)
  return |d₀| ≤ |d₁₈₀| ? d₀ : d₁₈₀   // минимум по модулю, знак сохраняется
```

**`allowFlip = true`** — деталь симметрична по ворсу (можно перевернуть на 180°).
При `allowFlip = true` результат всегда ∈ (−90°, 90°].

**Инвариант**: `|computeNapDeviation(...)| ≤ 90°` при `allowFlip = true`.

> **Исправленный дефект (2026-05-16):** прежняя реализация использовала `Math.min(d₀, d₁₈₀)` —
> алгебраический минимум знаковых значений. При `target=0°, rotated=10°, flip=true`
> это давало −170° вместо корректных 10°. Исправлено на минимум по модулю.

## 5. Фрагментация зоны (`server.js`, `src/modes/`)

Фрагментация разбивает зону (произвольный полигон) на набор фрагментов —
непересекающихся полигонов, объединение которых совпадает с зоной.

Общая схема для всех режимов:
```
1. Построить набор «шаблонных» полигонов, покрывающих bounding box
2. Пересечь каждый шаблон с зоной: fragment = intersect(template, zone)
3. Отфильтровать фрагменты меньше minAreaMm2
4. (при gap=0) Заполнить остаток: fillRemainderIntoFrags
```

### 5.1 Продольно-поперечная (`generateRegularFragments`)

Зона разбивается сеткой `rows × cols`.

**Стратегия `core_grid`** (продольный режим):
- Вертикальные разрезы: равномерно по всему bbox — `xᵢ = minX + i · W/cols`
- Горизонтальные разрезы: равномерно — `yⱼ = minY + j · H/rows`
- Результат: ячейки одинакового размера `(W/cols) × (H/rows)`

**Стратегия `core_overlap`** (со смещением к ядру):
- Вертикальные разрезы адаптируются к форме зоны через **scanline**:
  13 горизонтальных сечений определяют «рабочую ширину» зоны,
  квантили 0.75/0.25 дают левую/правую границу ядра.

**Scanline (пересечение горизонтали с полигоном):**

Для каждого ребра `(A, B)` — если горизонталь `y` пересекает ребро:
```
t = (y − Ay) / (By − Ay)
x_cross = Ax + t · (Bx − Ax)
```
Все пересечения сортируются; пары `[x₂ᵢ, x₂ᵢ₊₁]` — интервалы внутри полигона.

### 5.2 Диагональная и ёлочка (`generateDiagonalFragments`)

**Диагональная (axisCount=0):**

Линейное координатное преобразование:
```
u = y − slope · x,    где slope = tan(angleDeg)
```
Полосы определяются как `{(x,y) : band·step ≤ u < (band+1)·step}`.
Полигон полосы строится через `clipPolygonByBand` (Sutherland–Hodgman).

**Ёлочка (axisCount > 0, herringbone):**

V-образное преобразование относительно оси `axisX`:
```
u = y − orientation · slope · |x − axisX|
```
Полосы симметричны относительно оси, образуя «ёлочку».

### 5.3 Радиальная (`generateRadialFragments`)

Зона делится на `ringCount` концентрических колец × `sectorCount` угловых секторов.

**Секторный полигон** — кольцеобразный сектор (аннулярный сектор):
```
outer arc: (r₁·cos(a), r₁·sin(a)),  a ∈ [a₀, a₁]
inner arc: (r₀·cos(a), r₀·sin(a)),  a ∈ [a₁, a₀]  (обратно)
```

**Площадь аннулярного сектора:**
```
S = ½ · (r₁² − r₀²) · (a₁ − a₀)
```

**Параметры:**

| Параметр | Смысл |
|---|---|
| `ringCount` | число концентрических колец |
| `sectorCount` | число угловых секторов |
| `innerRadiusMm` | радиус центрального выреза |
| `rotationDeg` | угол поворота всей сетки |
| `centerMode` | `auto` (центр bbox) или `manual` |

## 9. Компьютерное зрение: определение направления ворса (`furlab-access`)

`furlab-access` решает задачу, предшествующую раскрою: **автоматически определить
направление ворса на фотографии куска меха**. Результат — угол в градусах,
который затем записывается в базу как `napDirectionDeg` детали.

### 9.1 Пайплайн

```
Фото меха (RGB)
      │
      ▼
Яркостная маска        ← BT.601: L = 0.299R + 0.587G + 0.114B
Порог 2-го перцентиля  ← тёмные пиксели = основание ворсинок
      │
      ▼
Морфология             ← эрозия → дилатация (открытие)
Связные компоненты     ← BFS с 8-связностью
      │
      ▼
PCA на компоненте      ← матрица ковариации → собственные значения
Главная ось            ← угол главного собственного вектора
      │
      ▼
Скоринг кандидатов     ← elongation · length · darkness · center
Лучший детектор        ← взвешенный ансамбль нескольких методов
      │
      ▼
napDirectionDeg        ← угол ∈ [0°, 360°)
```

### 9.2 Преобразование в яркость

Стандарт BT.601 (SDTV):
```
L = 0.299·R + 0.587·G + 0.114·B
```

### 9.3 Морфологические операции (`maskPrimitives.js`)

Работают на бинарных растровых масках `Uint8Array[w·h]`.

**Эрозия** (структурный элемент — квадрат со стороной `2r+1`):
```
Erosion[x,y] = 1  ⟺  ∀(dx,dy): |dx|≤r, |dy|≤r → mask[x+dx, y+dy] = 1
```
Убирает тонкие выступы и шум.

**Дилатация:**
```
Dilation[x,y] = 1  ⟺  ∃(dx,dy): |dx|≤r, |dy|≤r ∧ mask[x+dx, y+dy] = 1
```
Заполняет разрывы.

**Открытие** = Erosion ∘ Dilation — стандартная операция удаления шума
с сохранением формы крупных объектов.

### 9.4 Связные компоненты (`maskPrimitives.js`)

BFS (обход в ширину) с 8-связностью (включая диагонали):

```
queue = [seed]
while queue not empty:
    (x, y) = queue.pop()
    component.add((x, y))
    for (dx, dy) in {-1,0,1}²\{(0,0)}:
        if mask[x+dx, y+dy] and not visited:
            queue.push((x+dx, y+dy))
```

### 9.5 PCA для определения главной оси (`lineDetectionDetectors.js`)

Для компоненты из n пикселей `{(xᵢ, yᵢ)}`:

**Центроид:**
```
cx = (1/n) Σ xᵢ,    cy = (1/n) Σ yᵢ
```

**Матрица ковариации 2×2:**
```
Cxx = (1/n) Σ (xᵢ−cx)²
Cyy = (1/n) Σ (yᵢ−cy)²
Cxy = (1/n) Σ (xᵢ−cx)(yᵢ−cy)
```

**Собственные значения** (аналитически для 2×2):
```
tr  = Cxx + Cyy
det = Cxx·Cyy − Cxy²
disc = √(tr²/4 − det)

λ₁ = tr/2 + disc     (главная ось)
λ₂ = tr/2 − disc     (поперечная ось)
```

**Угол главной оси:**
```
θ = ½ · atan2(2·Cxy, Cxx − Cyy)
```

Единственный аналитически точный вариант для 2×2 — `eigFromCov()` в коде.

**Elongation** (вытянутость):
```
elongation = λ₁ / λ₂
```
Для линии ворса ожидается `elongation >> 1`.

**Длина компоненты вдоль главной оси:**
```
u = (главный единичный вектор) = (cos θ, sin θ)
tᵢ = (xᵢ−cx)·ux + (yᵢ−cy)·uy    ← проекция на главную ось
length = max(tᵢ) − min(tᵢ)
```

### 9.6 Скоринг кандидата (`analyzeDarkComponent`)

Взвешенная сумма нормализованных признаков:

| Признак | Формула | Вес |
|---|---|---|
| `lineLike` | `clamp((elongation − 2.2) / 18, 0, 1)` | 0.30 |
| `darkLike` | `0.6·clamp((170−meanL)/90) + 0.4·clamp((145−p10L)/80)` | 0.25 |
| `widthLike` | `clamp(1 − |strokeWidth−3.5|/3.5, 0, 1)` | 0.20 |
| `lengthLike` | `clamp((length−10)/120, 0, 1)` | 0.15 |
| `centerLike` | `1 − dist(centroid, image_center)` | 0.10 |

`strokeWidth = n / length` — средняя толщина компоненты в пикселях.

`p10Lum` — 10-й перцентиль яркости пикселей компоненты (самые тёмные).

### 9.7 Нормализация контуров деталей (`access_normalize_piece_contours.js`)

Независимая реализация формулы Гаусса для знаковой площади:

```
S_signed = ½ · Σᵢ (xᵢ·yᵢ₊₁ − xᵢ₊₁·yᵢ)
```

`S_signed > 0` — обход против часовой стрелки (CCW).
`S_signed < 0` — по часовой (CW).

Используется для:
- Вычисления площади `areaMm²`
- Приведения ориентации контура к стандартной (CCW)
- Зеркального отражения при необходимости
