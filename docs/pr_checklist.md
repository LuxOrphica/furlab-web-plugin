# PR Checklist (FurLab)

## Перед открытием PR
- [ ] Изменение покрыто тестом (или обосновано, почему тест не требуется).
- [ ] Нет деградации существующего поведения.
- [ ] Обновлена документация/контракт, если изменилось поведение.

## Команды (обязательно)
- [ ] `npm run encoding:check`
- [ ] `npm run lint` (если доступно в проекте)
- [ ] `npm run typecheck` (если применимо)
- [ ] `npm test`
- [ ] `npm run modes:testcases` (для затронутых режимов)
- [ ] `npm run ui:smoke` (если затронут UI)

## Качество изменений
- [ ] Нет "god diff" (разумный размер PR).
- [ ] Нет случайных рефакторингов вне задачи.
- [ ] Нет silent fallback и скрытых side effects.
- [ ] Соблюдены контракты режимов и инварианты.

## Безопасность и стабильность
- [ ] Нет утечек секретов, токенов, паролей.
- [ ] Не снижены пороги безопасности и quality-gates.
- [ ] Есть план отката для рискованных изменений.

## Self-test отчет в описании PR
Добавьте блок:

```md
Commands I ran:
- npm run encoding:check
- npm test
- ...

Result:
- PASS/FAIL

Manual scenarios:
- <scenario 1>: PASS/FAIL
- <scenario 2>: PASS/FAIL
```
