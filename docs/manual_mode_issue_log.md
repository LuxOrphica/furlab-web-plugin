# Ручной режим: журнал препятствий и выводов

Файл для фиксации проблем, которые проявляются при ручной сборке и указывают на слабые места алгоритма/UX.

## Формат записи

- `Дата`
- `Симптом`
- `Где проявилось`
- `Причина (гипотеза/подтверждено)`
- `Статус`
- `Комментарий`

## Записи

### 2026-03-06

1. Симптом: нельзя было поставить второй кусок после первого дропа.
- Где проявилось: ручной режим, dnd из лотка.
- Причина: блокировка цепочки дропа, если активный кусок в `tiny_fragment`/некорректном состоянии.
- Статус: исправлено.
- Комментарий: дроп больше не блокируется; при `gain>0` идет автофиксация, при `gain=0` активный кусок заменяется новым.

2. Симптом: первый кусок исчезал при дропе второго.
- Где проявилось: ручной режим, последовательный dnd.
- Причина: один активный незакоммиченный кусок перезаписывался новым.
- Статус: исправлено.
- Комментарий: добавлена автофиксация предыдущего активного куска (если есть прирост), иначе безопасная замена.

3. Симптом: `Commit placement` иногда не добавлял кусок.
- Где проявилось: ручной режим, после dnd.
- Причина: `lastMetrics` могли не успевать обновиться к моменту коммита.
- Статус: исправлено.
- Комментарий: перед коммитом выполняется принудительная оценка, если метрики отсутствуют.

4. Симптом: диапазоны в лотке выглядели нелогично.
- Где проявилось: секции `Большие/Средние/Малые`.
- Причина: смешение критериев и нечитаемая шкала.
- Статус: исправлено.
- Комментарий: диапазоны приведены к понятным и непересекающимся: `<=X`, `X-Y`, `>=Y` см.

5. Симптом: перегруженный Step 2 в ручном режиме.
- Где проявилось: модал шага 2.
- Причина: показывались solver-метрики, не нужные при ручной работе.
- Статус: исправлено.
- Комментарий: в ручном режиме оставлен минимальный UI + блок подсказок.

## Открытые наблюдения

- При ручной сборке оператор может собрать адекватное решение, а автоматический режим часто дает слабый результат.
- Вывод: базовая геометрия пригодна; зона риска — стратегия автоподбора и управление фазами выбора.
